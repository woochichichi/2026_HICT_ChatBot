### B. 기술 질문
| 질문 | 답변 |
|------|------|
| **"AI를 어떻게 학습시켰나?"** | 학습(파인튜닝)이 아니라 **검색 기반(RAG)**입니다. 질문이 올 때마다 편람에서 관련 문서를 찾아 AI에게 건네 답하게 합니다. 그래서 편람이 바뀌면 문서만 다시 넣으면 됩니다. |
| **"검색은 어떻게? 왜 정확한가?"** | 질문을 의미 좌표(벡터)로 바꿔 유사 문서를 찾고, **키워드(BM25)와 의미(벡터)를 합쳐** 정확도를 높였습니다. '양도세'로 물어도 '주식 양도 세금' 문서를 찾습니다(자체 hit@3 100%). |
| **"검색 정확도(hit@3)가 정확히 뭔가?"** | 검색이 **정답 든 문서를 상위 몇 개 안에** 찾는지를 잰 값입니다. 자체 30문항에서 정답이 **검색 상위 3개 안에 든 비율**이 100%(hit@3)입니다. '찾기' 단계 품질이고, 답을 쓰는 정확도(80%)와는 별개입니다. |
| **"기존 RAG와 뭐가 다른가?"** (← 차별점) | 기본 RAG는 '**의미 벡터 검색 한 번**'입니다. 우리는 ① 제목/본문을 **따로 검색해 합치고** ② 동의어 별칭을 **Max Pooling(최댓값)**으로 공정 처리하고 ③ **키워드(BM25)+의미(벡터)를 융합(Hybrid)** 하고 ④ **동의어 사전**으로 보정합니다. 이 4개가 서로 빈틈을 메워 hit@3 100%가 됩니다. 하나만 꼽으면 **③ 키워드+의미 융합**이 핵심 차별점입니다. |
| **"상위 몇 개 중 어떻게 답을 고르나?"** | 1개를 고르는 게 아니라 **상위 5개를 전부 AI에게** 주고, AI가 그 안에서 관련 내용으로 답을 쓰며 출처를 인용합니다. |
| **"답변 80%에서 남은 20%는?"** | 검색(찾기)은 100% 성공이고, **AI가 답을 쓰는(생성) 단계**에서 핵심을 빠뜨린 경우입니다. 검색이 아니라 작성 문제라 **프롬프트(지시문) 개선**으로 올라갑니다. |
| **"DB·임베딩 모델은 왜 그걸 골랐나?"** | DB는 **ChromaDB**(설치 한 줄 + 폴더 복사 = 폐쇄망 이식 최강), 임베딩은 **bge-m3**(한국어+영문+코드 혼합에 강한 다국어 로컬 모델, MIT 무료). 둘 다 **폐쇄망 이식성** 기준으로 선택. |
| **"채점·추천멘트는 누가 만드나?"** | AI(LLM)가 편람의 정답을 기준으로 점수·근거·누락 항목·모범답변, 그리고 고객 응대 추천 멘트까지 생성합니다. |
| **"데이터 처리 과정은?"** | 수집(HTML) → 파싱(제목·문단·표·공지 블록) → 조각내기 → 임베딩(bge-m3) → ChromaDB 저장. 매일 배치는 **변경분만**(내용 해시 비교) 다시 처리합니다. |
| **"위키 HTML은 어떻게 가져왔나?"** | 별도 API나 DB 접근 권한 없이, **사람이 브라우저로 보는 화면 그대로**를 프로그램이 받아왔습니다. PowerShell이 위키 주소로 HTTP 요청을 보내면서 **Windows 통합 인증**으로 자동 로그인되고, 위키의 '페이지 트리(목차 구조)'를 따라 하위 페이지를 모은 뒤 각 페이지 HTML을 통째로 내려받아 저장합니다. |
| **"통합 인증이 뭔가?"** | 사내 PC에 **이미 로그인된 Windows 계정 자격을 그대로** 써서, 별도 아이디·비밀번호·OTP 입력 없이 자동으로 인증되는 방식(SSO, 싱글 사인온)입니다. 자주 바뀌는 인증번호를 매번 넣을 필요가 없고, **그 계정이 볼 수 있는 페이지만** 가져옵니다(권한 그대로). |
| **"다른 분야·시스템에도 똑같이 되나?"** (← 핵심 관심사) | **네, 핵심 패턴이 그대로 재사용됩니다.** 사내망에서 HTTP로 접근되고 같은 통합 인증을 쓰는 시스템이면 — 게시판·그룹웨어·파일서버 등 — 동일하게 받아옵니다. 시스템마다 '주소·페이지 구조'만 다르니 **그 부분(수집기)만** 교체하고, 형식이 다르면(예: PDF) **해석기(파서)만** 추가합니다. 그 뒤 조각내기·임베딩·검색·답변은 **그대로**입니다. |

## 🔧 (심화 참고) 우리 검색이 기본 RAG와 다른 점

> "기존이랑 뭐가 다르냐"를 깊게 캐물을 때 펼치는 자료. 기본 RAG = **의미 벡터 검색 한 번**.

| # | 우리 방법 | 왜 (기본 RAG의 약점 보완) |
|---|---|---|
| ① | **제목/본문 2개 컬렉션** 따로 검색 후 가중 병합(0.5+0.5) | 제목은 '무슨 주제', 본문은 '구체 사실' — 둘 다 봐야 정확 (사람인 HR챗봇 87% 사례 참고) |
| ② | **동의어 제목 여러 개 + Max Pooling**(최댓값) | '양도세=주식 양도 세금=양도소득세'처럼 별칭을 많이 달아도, 합산 아닌 **최댓값**이라 별칭 수가 점수를 부풀리지 않음 |
| ③ | **Hybrid: 키워드(BM25) + 의미(벡터)를 RRF로 융합** | 벡터만 쓰면 'ELW·[11124]·K-OTC' 같은 정확한 용어·코드를 놓침. 키워드가 그걸 잡고, 벡터가 '미국주식→해외주식' 의미를 잡음 — **상호 보완** |
| ④ | **동의어 사전 확장**(검색 쿼리만, 파이썬 사전) | '미국 주식'으로 물으면 '해외주식' 문서를 못 찾는 **동의어 미스**를 보완 |

> **점수 방식 차이**: ①② = 유사도 **점수**로 합침 · ③ = 점수 척도가 달라 **등수(RRF)**로 융합 · ④ = 검색 전 **질문 확장**(점수 아님). → 단일 벡터는 '점수 1개로 정렬', 우리는 '여러 신호를 점수+등수로 겹쳐 빈틈 제거'.
