증권 상담원 AI 코치

“물어보면 답하고, 답하면 채점하는 AI”

상담원이 300초 걸려 찾던 답이 3초에 나옵니다.
무엇을 만들었고, 얼마나 좋아지는지 — 5분에 핵심만.

01 · 무엇을 만들었나

고객 질문 1건에 120~300초 —
상담원이 매번 업무 편람을 뒤진다

고객 질문 1건에 답하는 시간
120~300초
숙련 상담원도 매번 편람을 뒤져야 한다
신입이 혼자 응대할 수 있기까지
약 3개월
집합교육 + 선배 청취·동석 — 선배 부담이 크다
병목편람 지식이 사람 머릿속에만 + 고객은 두서없이 묻는다(상담원이 ‘무슨 편람 항목인지’까지 해석) → 편람을 알고, 고객 말 그대로 받는 AI를 만들면?

※ ‘120~300초’은 추정 — McKinsey·IDC(직장인이 정보 검색에 근무시간 20~30% 소비)를 1일 38콜로 환산 시 콜당 150~230초 (콜센터는 검색에 최대 47%라는 조사도 있음)

01 · 지금은 어떻게

지금은 이렇게 합니다 — 사람이 편람을 뒤진다

고객이 물으면 상담원이 직접 편람을 검색합니다. 어느 페이지에 있는지는 경험에 달렸고, 그동안 고객은 기다립니다. (자동 재생)

01 · 한눈에

편람을 아는 AI 동료 — 300초를 3초로

상담원이 편람을 뒤지던 일을 AI가 대신한다. 무엇을 만들었고, 누구에게 얼마나 좋아지는지부터.

상담사 · 속도

편람 검색 120~300초 → 3초 + 출처

신입 · 교육

집합·청취·동석 교육 부담 감소

상담사 · 품질

출처·추천멘트로 응대 품질 균일

핵심분기 약 3 M/M 절감(연구 기반) · 검색 정확도(hit@3) 100% · 답변 정확도 80% — 사내(폐쇄망)에서 작동

※ hit@3 = 정답이 검색 상위 3개 안에 든 비율  ·  약 3 M/M = 39명 × 검색시간(근무의 20~30%, McKinsey·IDC) × 53일 × 35%↓(콜센터 연구) ÷ 분기 480h (= 2.7~3.8 범위, 임의 가정 없음)

시연 · 챗봇 모드

같은 고객 질문, 이번엔 챗봇이 답한다

조금 전 상담원이 해석·검색하던 그 모호한 질문. 고객 말을 그대로 넣으면 → 편람 출처 + 답변 + 고객에게 읽어줄 추천 멘트까지 한 번에 (자동 재생)

시연 · 훈련(코치) 모드

훈련(코치) 모드 — AI가 채점

AI 고객이 질문 → 신입이 답변 → 점수 근거 + 모범답변 (자동 재생)

왜 믿을 만한가

폐쇄망에서 끝까지 — 외부로 안 나간다

검색·임베딩·DB는 전부 사내에서 동작. 외부로 나가는 건 답변 생성 1곳뿐이고, 그것도 사내 모델로 교체 예정.

🔒 폐쇄망 사내(native)

검색·임베딩·DB 전부 사내 — 외부는 답변 생성 1곳(교체 예정)

🌐 범용 — 수집 방식 그대로

사내 위키를 HTTP로 통째 수집(Windows 인증) — 게시판·파일서버·PDF도 같은 방식 + 파서만 추가

📊 검증됨

검색 hit@3 100% · 답변 80% · 틀려도 출처로 확인

핵심금융사 폐쇄망 통과 조건을 이미 충족 — 도입 시 바꿀 건 답변 모델 1곳뿐

※ 실제 HTML 수집: PowerShell이 사내 위키에 Windows 통합 인증으로 접속 → 페이지 트리를 순회하며 각 페이지 HTML을 통째로 받아 저장. 별도 로그인·API 없이, 게시판·파일서버도 같은 방식.  ·  hit@3 = 정답이 검색 상위 3개 안에 든 비율
