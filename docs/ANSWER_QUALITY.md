## ✅ 적용된 개선

### 1. 검색 쿼리 동의어 확장 (api-spec v11, 2026-06-16)

- **문제**: "미국 주식 하려면…"이 정답 문서 "(1) 해외주식 매매 개요 및 제도"를 top5에 못 올려 LLM이 "편람에 없음"으로 답함. ("해외주식 하는법"은 0.658로 정상 — 같은 의미인데 표현 차이로 갈림.) 원인은 LLM 거부가 아니라 **retrieval 미스**.
- **방법**: 검색 직전 질문에 도메인 동의어를 덧붙임(`backend/services/query_expand.py`). 임베딩·BM25 쿼리에만 적용, LLM에 가는 질문 원문은 불변 → 표시·출처 영향 없음. 순수 파이썬 사전(결정적·오프라인·지연 0).
- **근거/검증**: "미국 주식"에서 정답 문서가 top5 밖 → **[2] 0.634**로 진입, 정상 답변 확인. "일본주식 사고싶어"도 [1]로 진입.
- **한계**: 사전에 손으로 넣은 표현만 커버. "엔비디아 사는법", "ETF" 등은 여전히 미스 → 일반화는 백로그(아래 B).
- **토글**: `QUERY_EXPANSION_ENABLED`(기본 on)

### 2. confidence 임계값 bge-m3 재보정 (api-spec v12, 2026-06-16)

- **문제**: 임계값 0.85/0.70은 **Gemini(3072차원) 기준**. bge-m3는 max가 0.78이라 `high`가 절대 안 뜨고 정답도 대부분 `low`로 깔려 신뢰도 표시가 무의미.
- **방법**: bge-m3 점수 분포 기준 **high≥0.70 / medium≥0.50**로 재보정. 임계값을 config로 분리(`CONFIDENCE_HIGH/MEDIUM_THRESHOLD`)해 provider 교체 시 재튜닝 가능.
- **근거/검증**: 측정 분포상 0.70↑ 9건 / 0.50~0.70 14건 / 0.50↓ 7건. 실제 호출에서 "비대면 계좌개설" high(0.746), "해외/미국주식" medium(0.63~0.66), "점심 메뉴 추천"(범위 밖) low(0.491)로 라벨이 의미 있게 갈림.

### 3. 출처 페이지 중복 제거 — 페이지당 청크 캡 (api-spec v13, 2026-06-16)

- **문제**: "해외주식 하는법"의 출처가 **표시 5건인데 고유 페이지는 2건**. 같은 배당옵션 FAQ 페이지(`pageId=23665618`) 청크가 4번 중복 노출 → 출처 패널 도배 + LLM 컨텍스트 다양성 저하.
- **방법**: `RAGService.search()` 결과 선택 시 **페이지당 최대 `PER_PAGE_CHUNK_CAP`개(기본 1)** 청크만 채택하고 다른 페이지에 자리 양보. **컨텍스트 단계에서** 줄여 citation `[n]` ↔ 출처 순번 1:1 정합 유지.
- **근거/검증**: 동일 질문 출처가 **5건 표시 / 고유 5페이지**로 정리. 정답 페이지([2] 해외주식 매매 개요)는 유지되고 citation 정합 확인.
- **트레이드오프**: 캡=1이라 긴 편람 한 페이지가 2청크 필요한 경우 일부 손실 가능 → 필요 시 `PER_PAGE_CHUNK_CAP=2`(단, 출처에 동일 페이지가 다시 보일 수 있음, 번호 정합은 유지).

## ❌ 폐기한 시도

### A. 저점수 컨텍스트 절대임계 필터 (2026-06-16 검토 후 폐기)

- **아이디어**: top_score가 일정값 미만이면 컨텍스트를 LLM에 안 넣어 할루시네이션 방어.
- **폐기 근거**: bge-m3 점수 분포상 **범위 밖 질문이 실제 정답 질문보다 점수가 높은 역전**이 발생.
  - 범위 밖 "점심 메뉴 추천": top **0.491**
  - 실제 정답 질문 q10/q28/q30: top **0.302~0.306** (정답 1위인데도)
  - 즉 0.50으로 자르면 **멀쩡한 정답 7개(0.30~0.41)를 죽이고** 정작 범위 밖(0.49)도 못 거른다.
- **결론**: 절대 유사도는 bge-m3에서 in/out-of-domain을 가르지 못함. 거부는 지금처럼 **LLM이 컨텍스트로 판단**하게 두고, 점수 기반이 필요하면 절대값이 아니라 **상대 마진/리랭커**로 접근해야 함(아래 C).

## 📋 백로그 (효과 예상, 미착수)

### ③ 데모데이 답변 생성 한도 대비 (운영, 우선순위 높음)

- **문제**: 임베딩은 로컬(bge-m3)로 무제한이지만 **답변 생성은 여전히 Gemini 무료 20회/일**. 2026-06-16 테스트 중 실제로 한도 소진되어 생성 차단됨("오늘 무료 한도 모두 사용"). 데모 중 연타하면 막힘.
- **대비책 후보**: (a) 데모 전용 유료 키 분리, (b) 데모 질문 답변 캐시(golden), (c) 로컬 생성 LLM(폐쇄망 실도입 방향과 일치).
- **결정 필요**: 데모 전에 (a)/(b)/(c) 중 택1.

### B. 동의어 사전 일반화 (검색 커버리지)

- 현재 수동 사전은 등록 표현만 커버. 일반화 후보: LLM 쿼리 재작성/확장, HyDE(가설 문서 생성 후 검색), 동의어 자동 추출.
- **비용**: 검색 전 LLM 1회 호출 → '첫 토큰 1초' 목표·생성 한도와 충돌. PoC 데모엔 사전으로 충분, 실도입 시 재검토.

### C. 상대 마진 / 리랭커 기반 신뢰도·필터

- 절대 임계(A) 대신, top1과 top2 점수 **마진**이나 cross-encoder **리랭커** 점수로 신뢰도/거부 판단. bge-m3 절대값 역전 문제를 우회 가능.
- api-spec 백로그 T2(Reranking)와 연계.

### D. 제목:내용 가중치 그리드 서치

- 현재 0.5:0.5 고정. `[5:5, 4:6, 3:7]` 그리드로 hit@1 최적화 (README 3주차 계획, 미완).
